# Context Budget Strategy

Different AI models have different context windows. This strategy defines tiered loading so agents with smaller contexts can still operate effectively.

## Tiers

### Tier S: Infinite (1M+ tokens)
- Models: Gemini Pro 1.5, Gemini Flash 1.5, Gemini 2.0+
- Load: all files in `LOAD_ORDER.md` Tiers 0-3, plus broad repository source context.
- Use when: whole-repo analysis, cross-cutting architectural refactors, deep codebase investigations, finding scattered hallucinated dependencies.

### Tier A: Full (200K+ tokens)
- Models: Claude Opus, Claude Sonnet, GPT-4 Turbo, Gemini Pro 1.5
- Load: all files in `LOAD_ORDER.md` Tiers 0-3
- Use when: deep architecture work, system evolution, multi-file refactors

### Tier B: Standard (32K-128K tokens)
- Models: DeepSeek Coder, Codex, GPT-4o, Gemini Flash
- Load: Tier 0 (operating contract) + Tier 1 (working state) + targeted Tier 2 files for the task domain
- Use when: feature implementation, debugging, review

### Tier C: Compact (8K-32K tokens)
- Models: smaller cloud models, medium local models (13B-34B)
- Load: fast-path only (AGENTS.md, PROJECT_PROFILE.md, WHERE_LEFT_OFF.md, TODO.md, FIXME.md, PLAN.md, PRODUCT_BRIEF.md)
- Use when: focused single-file edits, quick fixes, inline completion

### Tier D: Minimal (4K-8K tokens)
- Models: small local models (7B and below)
- Load: AGENTS.md + PROJECT_PROFILE.md + WHERE_LEFT_OFF.md only
- Use when: simple edits with minimal context needed

## Usage

### Automatic detection
Adapter files in `_system/host-adapter-manifest.json` can declare a `context_tier` field. The `emit-tiered-context.sh` script reads this to produce the appropriate load sequence.

### Manual override
```bash
bootstrap/emit-tiered-context.sh . --tier B
bootstrap/emit-tiered-context.sh . --model deepseek-coder
```

### In adapters
Adapters for context-limited models should reference this document and the fast-path in `_system/LOAD_ORDER.md`.

## Optional output compression (assistant replies)

Tiering above addresses **input** context. To reduce **output** tokens when the user explicitly opts in, use the Cursor skill `.cursor/skills/concise-communication/SKILL.md` (Caveman-inspired, host-neutral) and command `.cursor/commands/concise-session.md`. **Do not** enable this by default: it must not strip user requirements, safety steps, or handoff packets. See the skill for forbidden cases.

## Optional input file compression (human-edited prose only)

**Relationship:** `concise-communication` / `/concise-session` reduce **assistant output** tokens. **Input** compression shrinks long **user- or maintainer-authored** prose files. Tiered loading (`emit-tiered-context.sh`, this document) remains the **primary** safe lever; file compression is **opt-in** and second-line.

**Stop condition — do not default automation here:** If a workflow would routinely touch **generated host adapters** (for example `CLAUDE.md`, `CODEX.md`, `.cursorrules`) or **validate-system required** files, do **not** batch-compress those paths. They are regenerated by `bootstrap/generate-host-adapters.sh` or are contract surfaces; compressing them is **lost on regen** or can **break strict validation**.

**v1 allowlist (bootstrap-enforced):** Only paths under `docs/` or `notes/` with extensions `.md`, `.txt`, or `.rst`. Everything else is refused by `bootstrap/compress-context-file.sh`.

**Forbidden targets (non-exhaustive):** `_system/**`, `bootstrap/**`, `.cursor/**`, `.github/**`, `AGENTS.md`, root host adapter filenames, registry/ops/mobile/ai/packaging trees, and any path not under the allowlist above.

**Tooling:** Optional delegation to upstream [caveman-compress](https://github.com/JuliusBrussee/caveman/tree/main/caveman-compress) via `bootstrap/compress-context-file.sh`. Set `CAVEMAN_COMPRESS_HOME` or install the skill under `~/.claude/skills/caveman-compress`. The master AIAST repo may vendor a factory-only copy under `_TEMPLATE_FACTORY/third_party/caveman-compress` (not shipped to minimal app installs by default). Compression requires the `claude` CLI on `PATH` (upstream uses `claude --print`).

**Backup convention:** Upstream writes `path.original.md` next to the compressed file (same stem). **Rollback:** `git checkout -- path` or restore from `.original.md`.

**After any compress:** `./bootstrap/validate-system.sh . --strict`, `./bootstrap/check-system-awareness.sh .`, and human diff review (reject if code blocks, paths, or URLs drift).

**Cursor UX:** Command `.cursor/commands/compress-context.md` (`/compress-context`); procedural mirror `.cursor/skills/compress-context-input/SKILL.md`.

**How the Caveman-compress subsystem works (mechanics):** AIAST’s `bootstrap/compress-context-file.sh` is a **safety wrapper** (allowlist, denylist, dry-run, root check). When those gates pass, it runs upstream **caveman-compress** as `PYTHONPATH=<skill-root> python3 -m scripts <absolute-file-path>`. That invokes `scripts/cli.py` → `compress_file()` in `compress.py`: read the file, call **`claude --print`** with a prompt that compresses **natural language** only (preserve code fences, inline backticks, URLs, headings, paths), write **`basename.original.md`** next to the target, write the compressed text back to the original path, then **validate** compressed vs backup; on repeated failure it restores the original. **Without** the `claude` binary on `PATH`, compression cannot run—tiered loading and manual editing remain the fallback.

**When to use:** After **`bootstrap/emit-tiered-context.sh`** and loading fewer files still leave one **long human prose** file (e.g. design narratives in `docs/` or scratchpads in `notes/`) worth shrinking—and you can **review the diff**. Do **not** use for contracts, generated adapters, or files loaded every session as Tier-0 unless you have explicitly accepted the risk.

**How to invoke (operators):**
```bash
# From repo root; always dry-run first
./bootstrap/compress-context-file.sh . docs/YOURFILE.md --dry-run
./bootstrap/compress-context-file.sh . docs/YOURFILE.md
```
Same pattern with `notes/...`. In Cursor chat, use **`/compress-context`** and pass the relative path. See `_system/TROUBLESHOOTING.md` if the script refuses the path or reports missing caveman/`claude`.

## Rules

- Never truncate or summarize governance files to fit a smaller tier. Load fewer files instead.
- Tier D agents should still respect the core contract in AGENTS.md.
- Tier C and D agents should note in WHERE_LEFT_OFF.md that they operated with limited context.
- Full-tier agents (A) should verify work done by limited-tier agents when taking over.
