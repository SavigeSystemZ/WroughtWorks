# UI UX Review Playbook

## Review inputs

- `DESIGN_NOTES.md`
- `_system/DESIGN_EXCELLENCE_FRAMEWORK.md`
- touched screens, routes, or components

## Review for

- hierarchy and scannability
- layout coherence
- state coverage
- responsive behavior
- design intent and polish
- consistency with the established visual language

## Must-fix findings

- unclear primary workflow
- unreadable or structurally broken touched surfaces
- missing empty, loading, or error states
- obvious design drift inside touched areas
- generic-looking or accidental-feeling UI in a surface that should feel intentional

## Output format

- must-fix issues
- quality risks
- polish opportunities
- design notes that should be recorded for future sessions
