## Failure modes & recovery

### Instruction layer & precedence
- **Failure mode:** Agents follow the wrong/contradicting instructions or a host overlay overrides repo truth.
- **Recovery:** `bash bootstrap/detect-instruction-conflicts.sh . --strict`

### Context routing & load order
- **Failure mode:** Agent misses required context or loads the wrong files and acts on stale assumptions.
- **Recovery:** `bash bootstrap/check-system-awareness.sh .`

### Registry contract graph
- **Failure mode:** A managed file drifts, becomes an orphan, or a critical file loses validator coverage.
- **Recovery:** `bash bootstrap/generate-system-registry.sh . --write && bash bootstrap/check-registry-contract-graph.sh .`

### Integrity signing
- **Failure mode:** Tampered or corrupted managed file goes undetected, or a stale manifest blocks updates.
- **Recovery:** `bash bootstrap/verify-integrity.sh --generate --target . && bash bootstrap/verify-integrity.sh --check --target .`

### Validation & doctor
- **Failure mode:** Repo is structurally broken but work proceeds anyway.
- **Recovery:** `bash bootstrap/system-doctor.sh . --strict --heal --report`

### Scaffold & isolation
- **Failure mode:** Scaffold leaks factory/MOS surfaces downstream or contaminates another repo.
- **Recovery:** `bash bootstrap/validate-scaffold-output.sh . --profile standard --dry-run`

### App identity & definition
- **Failure mode:** A blank app repo turns into a random codebase, or the template is edited as if it were an app.
- **Recovery:** `bash bootstrap/check-app-definition-state.sh .`

### Project-local self-improvement
- **Failure mode:** A downstream improvement escapes its repo or a self-write damages the operating layer.
- **Recovery:** `bash bootstrap/check-local-self-improvement.sh .`

### Agent coordination & locks
- **Failure mode:** Two agents edit the same operating surface and lose each other's writes.
- **Recovery:** `bash bootstrap/reap-stale-leases.sh . && bash bootstrap/check-agent-locks.sh .`

### MCP project isolation
- **Failure mode:** MCP config or app-id from another repo bleeds into this one.
- **Recovery:** `bash bootstrap/check-mcp-project-isolation.sh .`

### Host adapters
- **Failure mode:** A host adapter drifts from the canonical contract and instructs an agent incorrectly.
- **Recovery:** `bash bootstrap/generate-host-adapters.sh . --write && bash bootstrap/check-host-adapter-alignment.sh .`

### Git mirror & discipline
- **Failure mode:** Branch sprawl, divergent main, or treating the remote as authoritative.
- **Recovery:** `bash bootstrap/check-git-discipline.sh .`

### Quality & evidence
- **Failure mode:** Quality silently regresses or success is claimed without evidence.
- **Recovery:** `bash bootstrap/score-quality-gates.sh .`

### Release readiness
- **Failure mode:** An unvalidated or unreproducible template state ships to the fleet.
- **Recovery:** `bash bootstrap/generate-release-packet.sh . && bash bootstrap/validate-release-packet.sh . --latest`
