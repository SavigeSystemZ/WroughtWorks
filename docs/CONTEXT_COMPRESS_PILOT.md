# Context compression pilot (eligible prose)

This file exists only to exercise the **opt-in** Caveman-style input compression workflow (`bootstrap/compress-context-file.sh` and `/compress-context`). It is **not** a governance or tiered-load contract file.

## Filler prose (natural language)

When you are deciding whether to compress a markdown file, remember that **tiered loading** remains the primary safe lever for input tokens. Compression is a second-line option for long human notes that are not regenerated by host-adapter scripts and not listed as required validation surfaces.

A thoughtful approach is to keep one authoritative longform document and link shorter task notes to it, rather than duplicating requirements in multiple places. That reduces drift and avoids the need for aggressive compression in the first place.

Reference link preserved for validation exercises: https://github.com/JuliusBrussee/caveman/tree/main/caveman-compress

### More prose

Documentation should stay reviewable. If a compression pass removes nuance from safety steps, rollback using `git checkout -- path` or the `*.original.md` backup written beside the file by upstream tooling.

Operational discipline matters: run `bootstrap/validate-system.sh . --strict` after any compress, then read the diff. Reject hunks that change code fences, paths, or URLs.
