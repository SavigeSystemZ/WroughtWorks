# `app/docs/` — Application documentation

Documentation **about the product**, for its users and maintainers.

- **What goes here:** product/user docs, API references, architecture
  decisions *for the app*, runbooks.
- **What does NOT go here:** meta-system documentation — that lives in
  `_system/`. Keep the two separate so the product's docs ship with the
  product.
- **When to use:** as soon as the app has behavior worth documenting.
- **Empty state:** `.gitkeep` means no app docs exist yet.
