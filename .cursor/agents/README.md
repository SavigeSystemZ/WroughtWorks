# Cursor Agents

These optional agent definitions provide role-focused overlays for Cursor-like tools that support subagents or delegation.

They mirror `_system/AGENT_ROLE_CATALOG.md`. `AGENTS.md` and `_system/` remain canonical.

`composer-lead.md` is optimized for Cursor **Composer** sessions that span multiple files.

`github-ops.md` is for **GitHub Actions, PRs, and merge-readiness** slices (pair with `.cursor/commands/github-session.md`).

See `_system/HOOK_AND_ORCHESTRATION_INDEX.md` for how delegated agents fit with rules, commands, and CI.
