# Packaging

This directory is generated by AIAST and is project-owned after scaffolding.

## Included manifests

- `appimage.yml` for AppImage-style self-contained bundles
- `flatpak-manifest.json` for the preferred sandboxed Linux path
- `snapcraft.yaml` for Snap packaging
- `*.desktop` launcher metadata shared by Linux packaging targets
- `signing/README.md` for release-signing notes

## Placeholder review

Review these values before publishing:

- application id: `__AIAST_APP_ID__`
- app name: `__AIAST_APP_NAME__`
- package version: `__AIAST_VERSION__`
- icon path
- executable path
- runtime dependencies

## Recommended first path

Flatpak is the deepest-documented path in this template because it offers a strong default sandbox model for desktop applications.

Because the manifest lives under `packaging/`, keep its source dir rooted at
`..` so repo-root build artifacts like `dist/<app>` stay visible during
packaging builds.

Validate each manifest in CI before publishing to Flathub, Snapcraft, or an AppImage release channel.
