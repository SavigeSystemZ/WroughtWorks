# Install Operations

These scripts are project-owned scaffolds generated by AIAST.

Multi-platform notes and Windows/macOS/mobile stubs live under `distribution/`
after runtime generation. See `_system/CROSS_PLATFORM_DISTRIBUTION_AND_INSTALLER_STANDARD.md`.

## Commands

- `install.sh` installs or repairs a Linux deployment foundation.
- `repair.sh` recreates missing directories, env files, and units without deleting user data.
- `uninstall.sh` removes installed wrappers, units, and app-owned directories.
- `purge.sh` is an explicit destructive wrapper around `uninstall.sh --purge-data --purge-config`.

## Behavior

- Detects OS, CPU architecture, and package manager.
- Prefers packaged artifacts when present under `dist/`.
- Falls back to source-style installation if no package artifact is available.
- Allocates and persists an available loopback port in `ops/env/.env`.
- Creates least-privilege service user and group for system installs.
- Generates hardened `systemd` units via `bootstrap/generate-systemd-unit.sh`.

Review `ops/env/.env.example` and `ops/install/lib/runtime-foundation.sh` before using these scripts in production.
