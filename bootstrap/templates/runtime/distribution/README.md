# Distribution (multi-platform)

Project-owned scaffold generated by AIAST. Keep **one subtree per platform**
under `platforms/` so agents and release pipelines never confuse Windows,
macOS, Linux, or mobile deliverables.

## Conventions

- **Linux server/desktop**: primary scripts live in `ops/install/`; this tree
  documents packaging (`packaging/`), systemd, and install flags.
- **Windows**: use `platforms/windows/Install.ps1` as a starting point; prefer
  WiX, MSIX, or winget for production when the team is ready.
- **macOS**: document pkg/Homebrew; keep signing and notarization notes in
  `platforms/macos/README.md`.
- **Android / iOS**: link to `mobile/`; ship through store or enterprise
  channels per `MOBILE_GUIDE.md`.

Authoritative requirements: `_system/CROSS_PLATFORM_DISTRIBUTION_AND_INSTALLER_STANDARD.md`.
